# コミット規約

## プレフィックス

| プレフィックス | 説明 | 使用例 |
|------------|------|--------|
| `feat:` | 新機能追加 | feat: ユーザー認証機能を実装 |
| `fix:` | バグ修正 | fix: ログイン時のセッション保存エラーを修正 |
| `docs:` | ドキュメント | docs: READMEにインストール手順を追加 |
| `style:` | フォーマット修正 | style: コードフォーマットを統一 |
| `refactor:` | リファクタリング | refactor: ユーザーサービスの構造を改善 |
| `test:` | テスト | test: ログイン機能のE2Eテストを追加 |
| `chore:` | その他の変更 | chore: 依存パッケージを更新 |
| `perf:` | パフォーマンス改善 | perf: データベースクエリを最適化 |

## 作業単位の例

### バグ修正の場合
```bash
fix: エラーメッセージの定数を定義
fix: バリデーション関数を修正
test: エッジケースのテストを追加
fix: コンポーネントに修正を適用
docs: 変更内容をコメントで説明
```

### 新機能追加の場合
```bash
feat: 新しいAPIエンドポイントを定義
feat: データモデルを作成
feat: サービス層のロジックを実装
test: サービスのユニットテストを追加
feat: UIコンポーネントを作成
test: コンポーネントのテストを追加
docs: API仕様書を更新
```

### リファクタリングの場合
```bash
refactor: 共通処理を抽出
refactor: 型定義を整理
refactor: 不要なコードを削除
test: リファクタリング後のテストを確認
style: インポート文を整理
```

## 注意事項

- **1コミット1目的**: 各コミットは1つの明確な目的を持つ
- **日本語で記載**: チームメンバーが理解しやすいように
- **具体的に**: 「修正」ではなく「何を修正したか」を書く
- **作業単位**: 機能単位ではなく、実際の作業単位でコミット
