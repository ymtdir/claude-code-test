# 要求事項: PR作成時の自動ラベル付与機能

## Issue #3の内容

### 概要
`/create-pr`コマンド実行時に、PRの内容に応じて適切なラベルを自動的に付与する機能を追加する

### 背景
現在の`/create-pr`コマンドはPRを作成するがラベルは付与しない。PRの種類（feat/fix/docs等）に応じたラベルを自動的に付けることで、PRの分類と管理がより効率的になる。

### 提案
pr-creationスキルに以下の機能を追加：
1. コミットメッセージのプレフィックスからPRの種類を判定
2. PR作成後に適切なラベルを自動付与
3. 以下のマッピングルールを適用：
   - `feat:` → `enhancement`ラベル
   - `fix:` → `bug`ラベル
   - `docs:` → `documentation`ラベル
   - `refactor:` → `refactor`ラベル
   - `test:` → `test`ラベル
   - `chore:` → `chore`ラベル

### 期待される効果
- PR管理の効率化
- 自動的な分類によるレビュープロセスの改善
- 手動でラベルを付ける手間の削減
- プロジェクト全体のPR管理の一貫性向上
