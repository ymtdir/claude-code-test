# DESIGN.md — ミニマル家計簿のデザインシステム

ワイヤーフレームバンドル（[Claude Design](https://claude.ai/design) からのハンドオフ）を要約したデザイン仕様。
原本は [docs/mockups/家計簿ワイヤーフレーム.html](docs/mockups/家計簿ワイヤーフレーム.html)（ブラウザで開いて視覚確認）。

## 1. デザイン意図

「**iOS 標準アプリと並べて違和感のない、必要最小限の家計簿**」を目指す。

- **iOS 純正準拠**: SF Pro / システムカラー / SF Symbols / 標準パターン（List / Picker / TabView / Sheet）を直接使う。独自 UI は持ち込まない
- **ミニマリズム**: 装飾を排し、家計管理の本質的な機能（記録・分類・把握・コントロール）のみ
- **完全手動入力**: 自動取り込みや OCR は実装しない。手で記録する体験そのものを重視するユーザーを想定
- **ライト / ダーク両対応**: iOS 標準ダークパレット（true black `#000` ベース）に揃える

## 2. デザイン原則（Do / Don't）

| ✅ Do                                                                    | ❌ Don't                              |
| ------------------------------------------------------------------------ | ------------------------------------- |
| システムフォント（SF Pro 自動選択）を使う                                | カスタムフォントを導入する            |
| `Color.primary` / `Color.secondary` 等のセマンティックカラーを基本とする | 独自色を多用する                      |
| SF Symbols でアイコンを表現する                                          | ラスター画像のアイコンを持ち込む      |
| 標準コンポーネント（`List`, `Picker`, `TabView`, `Sheet`）を使う         | 独自 UI を一から組む                  |
| 金額は `CurrencyFormatter`（純粋 struct）で `¥1,234` 形式に統一          | View 内に通貨フォーマットを直書き     |
| 純粋な `#000` を避け、ダークでも `#f2f2f7` をテキストに使う              | 黒背景に純白テキストを置く            |
| 角丸 10pt のカード + 0.5px 区切り線で iOS Settings 風に整える            | Material Design 風の影 / 立体感を出す |

## 3. 画面構造

ルートは `TabView` による 5 タブ構成。タブごとの主要画面と役割は次の通り。

| ID   | タブ           | 役割                        | 特徴的な構造                                                                                       |
| ---- | -------------- | --------------------------- | -------------------------------------------------------------------------------------------------- |
| S-01 | 入力           | 収支の手動記録              | 上: 種別セグメント・大きな金額表示・フィールドカード / 下: テンキー（**000 / 0 / ⌫**、小数点なし） |
| S-02 | カレンダー     | 月単位の俯瞰                | 上: 月カレンダー（各日に支出簡略表示）+ 月次サマリヘッダ / 下: 選択日の収支リスト                  |
| S-03 | レポート       | 月次集計                    | カード 3 並び（支出 / 収入 / 差額）+ ドーナツチャート + カテゴリ別内訳                             |
| S-04 | 予算           | 予算設定と消化状況          | 全体予算 + カテゴリ別予算（プログレスバー、超過時は赤 + ⚠）                                        |
| S-05 | 設定           | 設定ハブ                    | カテゴリ管理 / 固定費の設定の **2 項目のみ**                                                       |
| S-06 | 収支入力シート | 編集 / カレンダー経由の入力 | S-01 と同じレイアウトを `sheet` で表示                                                             |
| S-07 | カテゴリ管理   | カテゴリ CRUD               | `List`、設定からプッシュ遷移、使用中は削除不可                                                     |
| S-08 | 固定費の設定   | 固定費参照リスト            | `List`、Transaction の自動生成は **しない**                                                        |

### 主要なナビゲーション

- カレンダー日付セルの **シングルタップ**: 下半分が当該日の収支リストに更新
- カレンダー日付セルの **ダブルタップ**: 当該日を初期値とした入力シート（S-06）を起動
- 設定タブからは `NavigationStack` で S-07 / S-08 にプッシュ遷移

## 4. 主要コンポーネント

| コンポーネント           | 役割                                                              | 実装目安                                               |
| ------------------------ | ----------------------------------------------------------------- | ------------------------------------------------------ |
| **タブバー（5 タブ）**   | 全画面共通の下部ナビ。背景は半透明（すりガラス）                  | `TabView` + `.tabItem`、`Color.accentColor` で選択強調 |
| **月次サマリヘッダ**     | カレンダーとレポートで再利用。支出 / 収入 / 差額の 3 カード横並び | `HStack` + `RoundedRectangle` カード                   |
| **カレンダーグリッド**   | 7 列、各日に支出簡略表示。`9999` 超は `k` 表記                    | `LazyVGrid(columns: 7)`                                |
| **テンキー**             | S-01 / S-06 の入力。3×4 で 000 ボタン入り、小数点なし             | `LazyVGrid(columns: 3)`                                |
| **フィールドカード**     | カテゴリ / 日付 / メモ等の入力導線。iOS Settings 風               | `List` の `.insetGrouped` または `Form`                |
| **ドーナツチャート**     | レポートのカテゴリ別支出                                          | SwiftUI `Charts` の `SectorMark`                       |
| **プログレスバー**       | 予算消化状況。超過時は赤                                          | `ProgressView` をスタイルカスタマイズ                  |
| **アイコン付きリスト行** | 設定画面の項目                                                    | `Label` + 角丸背景アイコン                             |

## 5. デザイントークン（要点のみ）

詳細値はワイヤーフレームの [`wireframe-kit.jsx`](docs/mockups/家計簿ワイヤーフレーム.html) の `WF` / `WFD` を参照。

### カラー（用途ベース）

| ロール             | Light     | Dark      | 用途                   |
| ------------------ | --------- | --------- | ---------------------- |
| Primary text       | `#1d1d1f` | `#f2f2f7` | 本文・主要ラベル       |
| Secondary text     | `#5a5a5f` | `#a8a8ad` | 補足ラベル             |
| Background         | `#fafaf6` | `#000000` | 画面背景               |
| Card               | `#ffffff` | `#1c1c1e` | カード・リスト背景     |
| Accent             | `#ff6b3d` | `#ff7d54` | 選択日・強調アクション |
| Tint (system blue) | `#3a82f7` | `#0a84ff` | ナビ・リンク・編集     |
| Income             | `#34c759` | `#30d158` | 収入・予算消化未満     |
| Over budget        | `#ff3b30` | `#ff453a` | 超過警告               |

実装方針: `Assets.xcassets/Colors/` に Light / Dark の両値をセットしたカラーアセットを定義し、`Color("ink")` 等で参照。

### タイポグラフィ階層

| ロール                   | サイズ目安 | ウェイト  |
| ------------------------ | ---------- | --------- |
| 大見出し（画面タイトル） | 22pt       | 700       |
| 大金額表示（入力画面）   | 38pt       | 700       |
| カード金額               | 18pt       | 700       |
| 本文                     | 12〜14pt   | 400 / 500 |
| キャプション             | 9〜11pt    | 400       |
| ラベル小見出し           | 8〜9pt     | 600       |

フォントは SwiftUI の `.system()` に任せる（SF Pro 自動選択）。

## 6. 共通パターン

- **カード**: 角丸 10pt、背景は `Card` ロール、左右マージン 12pt、行間 0.5px の divider
- **ボタン**: ナビゲーション系（保存・編集・追加）はテキストのみ + `Tint` 色。ピル型 CTA は使わない
- **アニメーション**: 月切替は `.easeInOut(0.25)`、選択日変更は `.spring(response: 0.3)`
- **タブバー背景**: 半透明（`.ultraThinMaterial` 相当）でスクロール時の溶け込みを実現
